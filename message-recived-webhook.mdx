title: Webhook Ricezione Messaggio Ricevuto

---

Di seguito viene riportata la struttura dei dati contenuti nella chiamata POST che l’url indicato dal cliente riceverà in caso di ricezione di un messaggio sul telefono connesso a Trilly.

```bash json
{
  "body": stringa, <! – testo messaggio –>
  "from": TELEFONO_MITTENTE,
  "to":  TELEFONO_RICEVENTE,
  "media": URL MEDIA DOWNLOADABILE,
  "timestamp": "YYYY-MM-DDTHH:mm:SSZ" (fuso orario UTC),
  "real_timestamp": Campo timestamp con fuso orario locale che contiene l'orario di ricezione del messaggio effettivo, se presente
}
```

> Il campo body sarà sicuramente vuoto nel caso di invio di un media vocale. Il media vocale avrà estensione .aac

> Nel caso in cui venga inviata una posizione, sarà presente il campo “location”, che conterrà in formato json la posizione inviata.
